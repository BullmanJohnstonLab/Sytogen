CodonBias

CodonBias calculates strain-specific codon-usage patterns from an annotated bacterial genome file and provides the codon-bias information needed to guide synonymous edits during RM-silencing.

These patterns help ensure that sequence edits introduced by SyToGen remain biologically neutral and compatible with the host's translational machinery.

CodonBias Input

  • Bacterial genome in GenBank format (full genome annotation file containing all coding sequences from which codon frequencies can be calculated)

CodonBias Outputs

  • Codon-usage table (.csv) (codon counts and normalized usage frequencies for each amino acid)
  • Annotated GenBank file (.gbk)
  • Matched FASTA and GFF3 files (.fasta, .gff3)
  • Summary report (describing genome composition, codon-usage distributions, and any parsing notes)
  • Internal codon-preference metrics (used directly by SyToGen when selecting permissible synonymous edits)

About CodonBias

RM-silencing in coding regions relies heavily on synonymous substitutions: nucleotide changes that eliminate an RM motif without altering the amino acid sequence encoded by a gene. However, different synonymous codons are not used equally by different bacteria, which can differ widely in their preferred codons, often in ways that reflect their GC content, tRNA isoacceptor pools, and translational efficiency requirements.

Because codon usage varies across bacterial lineages, SyToGen must incorporate the codon-usage landscape of the specific organism of interest to ensure that any sequence edits made to genetic tools remain compatible with its native translational preferences. Without this c odon-usage information, edits introduced during RM-silencing may reduce expression, alter protein folding, or introduce unintended biological consequences (such as reduced transformation efficiency). CodonBias thus generates strain-specific codon-usage table directly from a bacterial genome annotation.

What does CodonBias do?

CodonBias extracts all coding sequences (CDS) from the provided genome file, counts every codon occurrence, and calculates normalized usage frequencies for each amino acid. It produces codon-usage tables and summary files that describe the quantitative codon-usage preferences of the strain-of interest.

The module also:

  • Handles multi-contig and multi-chromosome genomes
  • Resolves annotation inconsistencies in CDS feature tables
  • Excludes pseudogenes or truncated ORFs if appropriately annotated
  • Computes amino-acid-specific codon distribution ratios
  • Formats outputs for direct use by the SyToGen

These outputs ensure that all synonymous mutations proposed by SyToGen are made in accordance with the strain's codon-usage bias.
